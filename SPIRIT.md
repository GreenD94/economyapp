# Economy App — Spirit

## The vision in one sentence

This app is a finance expert you hired — not a spreadsheet you fill in.

## What that means

The user comes in with two facts: a goal and an income. The app does everything else. It calculates how much to save per month, how long it will take, and how much is left for each spending category. The user never has to do math or know finance theory to use this app.

## The app's personality

- Confident. It gives recommendations, not open-ended questions.
- Honest. It tells you if your goal is aggressive or realistic based on your numbers.
- Respectful of the user's time. Every screen has one job.

## Core beliefs

- Users who know nothing about finance can reach their savings goals with the right guidance.
- The best financial tool is one that removes decisions, not one that multiplies them.
- The app should feel like someone is looking out for you, not auditing you.

## Anti-patterns — things this app never does

- Asks the user a question the app can calculate itself.
- Shows raw numbers without context (e.g., "$580/month" without "that's 20% of your income").
- Lets the user configure something before they understand why it matters.
- Adds a feature because it's common in finance apps — only add it if it serves the hired-expert vision.

## Feature alignment test

Before building any new feature, ask:
1. Does this help the user save money or understand their finances better?
2. Does it reduce decisions or increase them?
3. Would a hired finance expert offer this, or is it just extra complexity?

If the answer to any of these is "no" or "it adds complexity," don't build it.

## What the app is NOT

- Not a general-purpose expense tracker.
- Not an accounting tool.
- Not a budgeting tool where the user sets up everything manually.
- Not a dashboard full of charts the user has to interpret.
